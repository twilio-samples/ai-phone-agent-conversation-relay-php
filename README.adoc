= AI Phone Agent Conversation Relay PHP

== Application Overview

The application is a fully functional AI phone agent written in PHP using {mezzio}[Mezzio] and {mezzio-swoole}[Mezzio-Swoole], the {openai-api}[OpenAI API] as the language model, and {deepgram-flux}[Deepgram Flux] for state-of-the-art speech recognition.

When someone dials your Twilio number, Twilio requests TwiML from the application's "/twiml" endpoint.
The endpoint returns a {conversation-relay-twiml-verb}[<ConversationRelay>] verb inside a {connect-twiml-verb}[<Connect>] block, which tells Twilio to open a [WebSocket] connection to your server and hand the call off to [Conversation Relay].

From that point on, Conversation Relay acts as the voice pipeline.
It transcribes everything the caller says using speech-to-text (STT) and sends the transcript to your WebSocket server as a JSON message.
Your server calls an AI model with the transcript, streams the response back over the WebSocket, and Conversation Relay synthesises each token into speech using text-to-speech (TTS), playing it to the caller in real time.

== Prerequisites

To run the app locally, you need the following:

* An {ngrok}[ngrok] account
* An {openapi-ai-key}[OpenAI API key]
* {composer}[Composer]
* {php}[PHP 8.4] or later
* {try-twilio}[A Twilio account] with a voice-capable phone number

== Quickstart

. Clone or download this repository
. Install the dependencies:
+
[source,bash]
----
composer install
----

. Rename the _.env.example_ file to _.env_
. Start ngrok listening on port 8080 and :
+
[source,bash]
----
ngrok http 8080
----
. Copy the Forwarding URL from the terminal output
+
image::https://github.com/twilio-samples/ai-phone-agent-conversation-relay-php/blob/56d8b2bf6da5f27c1cc9c2007833dc8e2bef1c44/docs/images/ngrok-terminal-output.png?raw=true["Terminal output from running ngrok with the Forwarding URL highlighted"]

. Configure your Twilio Account:
** Log into {twilio-console}[the Twilio Console]
** Navigate through menu:Phone Numbers[Manage > Active Numbers]
** Click your voice-capable phone number to open its configuration page
+
image::https://github.com/twilio-samples/ai-phone-agent-conversation-relay-php/blob/56d8b2bf6da5f27c1cc9c2007833dc8e2bef1c44/docs/images/twilio-console-voice-configuration-rendered-in-safari-on-macos.png?raw=true["The Twilio Console Voice Configuration form, loaded with the default settings"]

** Under the *Voice Configuration* section, set:
*** *A Call Comes In* to "Webhook" and enter the URL for your TwiML endpoint.
It will be your ngrok Forwarding URL, plus the suffix "/twiml", e.g., https://1234abcd.ngrok.app/twiml
*** Set the *HTTP method* to "HTTP POST"
** Scroll down to the bottom of the page and click btn:[Save Configuration]

. Set the Required Environment Variables
** Set the ngrok Forwarding URL's domain, minus the protocol, i.e., "https://", as the value of the `DOMAIN` variable in _.env_
** Set your OpenAI API key as the value of `OPENAI_API_KEY` in _.env_

. Start the Application
+
[source,bash]
----
DOMAIN=<ngrok domain> OPENAI_API_KEY=<your OpenAI api key> vendor/bin/laminas mezzio:swoole:start
----
+
This will start four workers listening on port 8080
+
NOTE: To stop the workers, run `vendor/bin/laminas mezzio:swoole:stop` in a separate terminal window or session.
